# Security Policy

## Supported versions

Security updates are applied to the latest active branch.

## Reporting a vulnerability

Please do not open public issues for security vulnerabilities.

Report vulnerabilities privately by contacting the maintainers and include:

- a clear description of the issue
- impact and affected endpoints/components
- step-by-step reproduction
- proof of concept (request/response examples)
- suggested remediation, if known

## Response targets

- Initial acknowledgment: within 72 hours
- Triage decision: within 7 days
- Fix timeline: based on severity and exploitability

## Disclosure process

- We validate and triage the report.
- We prepare and test a fix.
- We coordinate disclosure after a patch is available.
